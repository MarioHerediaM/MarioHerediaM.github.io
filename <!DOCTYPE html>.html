// DESCRIPCIÓN

Aplicación web que permite a usuarios registrados crear proyectos, organizarlos en tareas y hacer seguimiento del estado de cada una. Incluye sistema de autenticación propio y panel de administración.

// DESAFÍO PRINCIPAL

Primer contacto con el desarrollo backend. El desafío fue aplicar simultáneamente el patrón MVT, relaciones entre modelos con ForeignKey, migraciones, autenticación y enrutamiento de URLs — sin experiencia previa en ningún framework web.

// SOLUCIÓN PROPUESTA

Arquitectura de dos modelos relacionados: Proyecto vinculado al usuario autenticado y Tarea vinculada a su proyecto con sistema de estados. CRUD completo, protección CSRF y rutas restringidas por autenticación.

// APRENDIZAJES CLAVE

Comprender el ciclo completo de una solicitud web. Modelar relaciones con el ORM de Django. Aplicar autenticación real, protección CSRF y control de acceso. Gestionar cambios de esquema con migraciones.

// HABILIDADES APLICADAS

Modelado de datos · ForeignKey · Vistas basadas en funciones · Sistema de autenticación Django · Migraciones · Django Admin · Control de versiones con Git.

// ¿POR QUÉ ESTE PROYECTO?

Representa mi punto de inflexión como desarrollador. No es un tutorial — es un sistema que tuve que entender, romper y reconstruir varias veces. Refleja exactamente el stack que buscan empresas como LATAM Airlines y es la base sobre la que estoy construyendo mis próximas habilidades.
